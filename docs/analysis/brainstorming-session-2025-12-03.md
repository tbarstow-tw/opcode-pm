### Pattern Extraction

#### The "Comfort" Stack (Notion + Linear DNA)

**Visual Comfort Formula:**
- Dark background: #191919 (soft dark, not pure black)
- Text color: #E3E3E3 (warm white, not harsh)
- Font: Inter or similar screen-optimized typeface
- Line height: 1.6-1.8 (generous breathing room)
- Transitions: 200-300ms (smooth, not jarring)

**Emotional Design Insights:**
> "Typing in Notion is very calming. The font and color scheme for dark mode is unbelievably crisp and calming. Every color or font or interaction feels perfect."

**What Creates Calmness:**
- Zero lag between keystroke and render
- Subtle hover states (not jumpy)
- Clear but non-aggressive focus states
- Helpful error messages (not scary)
- Every micro-interaction harmoniously designed

#### The Safety Paradox Discovery

**Terminal Experience:**
> "Terminals feel like I could break something and have no idea how to fix it... powerful, delicate, and mysterious"

**Notion Experience:**
> "Notion feels like I can always fix an error I make... safe, calming, and sturdy"

**The Core Insight:**
- Terminal = Powerful + Delicate + Mysterious = Anxiety
- Notion = Safe + Calming + Sturdy = Confidence

#### The "Sturdiness" Pattern

**How to Make Software Feel Sturdy:**
1. **Undo/Redo everywhere** - Cmd+Z always works
2. **Trash before delete** - Nothing disappears immediately
3. **Version history** - Can always go back in time
4. **Visual confirmation** - See changes as they happen
5. **Soft failures** - Errors inform, don't crash
6. **Progressive disclosure** - Complexity hidden until needed

**Implementation for Opcode:**
- Replace `claude-code --delete-session abc123` with visual "Move to Trash" â†’ 30-day recovery
- Replace red error text with friendly explanations: "Oops, that workflow needs a project. Click here to choose one."
- Add Preview â†’ Confirm â†’ Execute â†’ Undo flow for all destructive actions

#### The "Precision" Pattern (Figma DNA)

**Figma's Precision Formula:**
- **Bottom toolbar** = All tools visible, spatially consistent
- **Left hierarchy** = See everything, click exactly what you want
- **Properties panel (right)** = See all values, no guessing

**Key Insight: Selection Over Description**
- Terminal: "I need to describe what I want" â†’ `find . -name "*.tsx"`
- Figma: "I'll click exactly what I want" â†’ *clicks component in hierarchy*

**For Opcode:**
- Visual hierarchy showing all agents, workflows, sessions
- Bottom toolbar with common actions always visible
- Click to select, not type to describe

#### The "Three Paths" Pattern

**Successful tools offer THREE ways to do everything:**
1. **Visual** - Click in UI (for newcomers)
2. **Keyboard** - Shortcuts (for power users)
3. **Search/Command** - Type when you know the name

**Terminals offer ONE way:**
- Type the exact command or fail

**For Opcode, every action needs three paths:**
- Visual: Click workflow in sidebar
- Keyboard: Cmd+R for run workflow
- Command: `@workflow/create-prd` in chat

#### The "Competence" Pattern (Linear DNA)

**What Makes Users Feel Competent:**
1. **Speed of accomplishing tasks** - Everything instant
2. **Knowing what to do next** - Smart defaults
3. **Not thinking about HOW, just WHAT** - Focus on goals

**Linear's Success Formula:**
- Keyboard-first but not keyboard-only (Cmd+K for everything)
- Speed - Everything happens instantly (<100ms)
- Minimal UI - Only shows what matters now
- Smart defaults - Guesses what you want next
